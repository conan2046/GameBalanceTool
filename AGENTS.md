## 六、本项目工程规则

项目根目录：

```text
D:\GameBalanceTool_V3.6.1
```

当前项目定位：

- Game Balance Tool V3.x
- 游戏数值模拟平台
- 核心模块包括属性、职业、境界、装备、经济、礼包、曲线库、养成树、ROI、多方案工程版本化。

### 工程协作原则

1. 修改前先读 README、docs/VERSION_HISTORY.md 和相关源码。
2. 不随意重构无关模块，优先小步修复、可验证交付。
3. 业务逻辑优先复用现有模块：
   - `ProjectState`
   - `project-versioning`
   - `curve-library`
   - `roi-strategy`
   - `simulator`
4. V3 模块仍与 `index.html` 旧内联逻辑共存，修复时必须兼顾旧 `onclick` 和 `window.xxx` 挂载。
5. 项目状态源必须统一，注意旧版 `let S` 与新版 `window.S` / `ProjectState` 的桥接。
6. 存档、导入、导出必须兼容：
   - `gbt-project`
   - `gbt-project-state`
   - v3 旧快照
   - v2.1 数据结构
7. 新增功能必须补基础验证，至少执行：

```bash
npm test
```

必要时执行：

```bash
Get-ChildItem -Recurse -File -Include *.js,*.mjs src,workers,libs,scripts,tests | ForEach-Object { node --check $_.FullName }
```

### 已知重点风险

1. `index.html` 仍很大，旧逻辑和 V3 模块双轨共存。
2. V3 模块依赖 ES Module、HTTP 服务和正确 MIME，不支持直接 `file://` 打开。
3. npm 依赖安装在当前环境可能出现 cache 权限或 npm 自身错误，优先使用项目内脚本和本地依赖。
4. Playwright 测试走项目内命令：

```bash
npm test
```

5. 开发服务器走：

```bash
npm run dev
```
