# GameBalanceTool V3.8.2 使用手册

## 1. 快速启动

### Windows 一键启动

双击项目根目录：

```text
start-gbt.bat
```

启动器会自动完成：

| 步骤 | 说明 |
|---|---|
| 检查端口 | 检查 `127.0.0.1:8080` 是否已能访问 |
| 启动服务 | 未启动时执行 `node scripts/static-server.mjs --port 8080` |
| 端口占用 | 如果 8080 被其他程序占用，会提示进程号，不会自动杀进程 |
| 打开网页 | 服务可访问后自动打开 `http://127.0.0.1:8080/` |

### 命令行启动

```powershell
cd D:\GameBalanceTool_V3.6.1
npm run dev
```

浏览器访问：

```text
http://127.0.0.1:8080/
```

不要直接双击 `index.html`。V3 模块依赖 ES Module 和正确 MIME，必须通过 HTTP 服务访问。

## 2. 页面模块总览

| 模块 | 用途 |
|---|---|
| 属性与权重 | 管理二级属性、基础值、战力权重 |
| 属性成长矩阵 | 查看属性成长和曲线结果 |
| 职业设定 | 配置职业、伤害类型、职业克制和 1v1 关系 |
| 境界设定 | 配置修为境界、炼体境界和境界差系数 |
| 装备系统 | 管理槽位、品质、精炼和装备收益 |
| 养成系统 | 配置养成分支、消耗规则、属性收益、曲线绑定 |
| 资源管理 | 维护资源、货币和消耗产出结构 |
| 经济产出 | 规划日常产出、资源投放和经济循环 |
| 礼包/商店 | 做礼包价值、商店定价和资源换算 |
| 战斗演算 | 对比不同职业、属性、公式下的战斗结果 |
| ROI 对比 | 模拟不同养成投资策略的回收效率 |
| 曲线库 | 维护线性、指数、对数、S 型等成长曲线 |
| 多方案工程 | 管理项目方案、版本快照和策略对比 |

## 3. 推荐使用流程

### 新项目建模

1. 在“属性与权重”定义核心属性和战力权重。
2. 在“曲线库”建立等级、消耗、收益曲线。
3. 在“职业设定”补齐职业、伤害类型和克制关系。
4. 在“境界设定”配置修为/炼体阶段。
5. 在“养成系统”绑定曲线、配置分支消耗和属性收益。
6. 在“经济产出”配置资源投放。
7. 在“ROI 对比”和“战斗演算”验证数值表现。
8. 使用“导出数据”保存工程 JSON。

### 调整已有项目

1. 先导入旧工程 JSON。
2. 确认属性、曲线、养成分支是否迁移完整。
3. 修改单个模块后先观察联动模块，例如曲线改动后检查 ROI 和养成明细。
4. 每次大改前导出一份带日期的 JSON 备份。

## 4. 导入、导出与存档

| 操作 | 说明 |
|---|---|
| 导出数据 | 导出当前工程 JSON，用于备份或跨设备传递 |
| 导入 | 导入已有 JSON，支持 v3 工程封包和部分旧结构迁移 |
| 浏览器存档 | 使用 IndexedDB/localStorage 持久化当前状态 |

建议每次完成一轮公式或经济参数调整后导出一次 JSON，文件名带版本和日期，例如：

```text
GBT_修仙项目_境界经济_v2026-06-26.json
```

## 5. 测试与验收

基础自动化测试：

```powershell

```


手动验收重点：

| 项 | 通过标准 |
|---|---|
| 启动 | `start-gbt.bat` 能打开网页 |
| 页面加载 | 控制台无红色模块加载错误 |

## 6. 常见问题

### 127.0.0.1 拒绝连接

原因通常是本地服务没启动。处理：

```powershell
cd D:\GameBalanceTool_V3.6.1
.\start-gbt.bat
```

### 8080 端口被占用

查看占用进程：

```powershell
Get-NetTCPConnection -LocalPort 8080 -State Listen
```

如果不是本项目服务，先关闭对应程序，再重新双击 `start-gbt.bat`。

### 双击 index.html 页面异常

这是预期限制。V3 模块依赖 ES Module，必须通过 `http://127.0.0.1:8080/` 访问。

### npm test 会自动启动服务吗

会。Playwright 配置会自动启动 `node scripts/static-server.mjs --port 8080`。

手动浏览器测试仍建议用 `start-gbt.bat`。
